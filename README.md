# dbt_data_quality

> **Реплицируемый проект Data Quality на dbt**  


## 🎯 Цель проекта
Централизованное выявление, классификация и мониторинг ошибок качества данных (в т.ч. в документах продажи — расхождения ставок НДС).

Проект спроектирован так, чтобы его можно было:
- Легко развернуть в любой среде
- Масштабировать на новые категории ошибок
- Интегрировать с внешними системами (DataLens, BI-инструменты)
- Использовать как законченный кейс в резюме

## 📐 Архитектура (высокий уровень)
Источники данных (Экстрактор 1С / БД)
↓
Staging + Intermediate (dbt)
↓
Marts / Data Quality (err_xx_yy)
↓
Единая витрина mart_all_errors
↓
DataLens / BI-дашборды
**Текущий фокус:** DBT-слой (модели, тесты, документация).  
Полная цепочка (получение сырых данных и выгрузка → DataLens) будет добавлена позже.

## Подпроекты Data Quality

### 07.02 — Документы с некорректной ставкой НДС

**Цель:** Выявление документов продажи, оплаты и возвратов с ошибочными ставками НДС.

**Текущий статус:** В разработке (реализуется первый микросервис — Чеки ККМ).

**Документация по подпроекту:**
- [Архитектура и этапы преобразования](docs/07_02_architecture.md)
- [Поток данных](docs/07_02_data_flow.md)
- [Соглашения по именованию моделей](docs/07_02_naming_conventions.md)
- [Бизнес-описание и правила ошибок](docs/07_02_business_description.md)

**Ключевые особенности:**
- 5-слойная модель преобразований (staging → prepare → enriched → errors → all_errors)
- Изоляция по типам документов (отдельные error mart'ы)
- Возможность независимого пересчёта отдельных типов документов



## 🛠️ Стек технологий (EtLT)

**Полная цепочка данных:**

### Extract + Load (E + L)
- **1С Экстрактор** — выгрузка сырых данных из 1С (шапки документов, табличные части, справочники)
- Целевые БД: ClickHouse

### Transform (T)
- **dbt Core 1.11.11**
  - Адаптеры: `dbt-postgres`, `dbt-clickhouse`
  - Слои: `staging` → `intermediate` → `marts/data_quality`
  - Стратегии: `table` + `incremental`
  - Тестирование и документация

### BI / Visualize
- **Yandex DataLens** — дашборды по ошибкам качества данных, фильтры, аналитика по периодам и ответственным
