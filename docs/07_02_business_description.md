# 07.02 — Документы с некорректной ставкой НДС

## Описание задачи
Выявление в системе 1С документов, подлежащих передаче в ОФД/ФНС (продаж, оплат и возвратов от клиента) и содержащих некорректные, согласно законодательству и учётной политике организации, на момент проверки, ставки НДС.


## Бизнес-ситуация:
**Правила применения ставки:**
- Ставка НДС документа = актуальной ставке НДС организации.
- Исключение: льготная ставка НДС (10%) у соответствующей номенклатуры, если у организации общая ставка (22%).
- Все ставки НДС - используются только основные (5%, 7%, 10%, 22%). Авансовые ставки (10/110% и аналогичные) в документах 1С - не используются.
- При возврате от клиента, ставка НДС в возврате = ставке НДС в исходном документе продажи (независимо от ставки организации на текущий момент).


### Цель DataMart:
Выявить все документы продажи/оплат/возвратов с ошибочными ставками НДС для контроля качества и исправлений.


### Описание DataMart 
Список ошибочных документов с указанием:
- даты и типа документа,
- наименования и ссылки на документ,
- категории ошибки,
- полей, подтверждающих ошибку,
- организации,
- ответственного (продавца) и его подразделения.


### Что считается ошибкой:
1. Ставка НДС документа или организации - Null/незаполнена.
2. Ставка НДС документа ≠ ставке НДС организации (в т.ч. Null/незаполнена).
3. Ставка НДС документа ≠ льготной ставке номенклатуры, при общей ставки НДС организации (22%).
4. Ставка НДС документа возврата ≠ ставке НДС документа продажи.


## Используемые источники данных (Bronze / Raw)

### Документы (факты):
- `check_kkm` - Чек ККМ (продажа)
- `sale_doc` - Реализация товаров и услуг (продажа)
- `retail_sale_report` - Отчёт о розничных продажах (продажа)
- `payment_card_operation` - Эквайринговая операция (оплата)
- `incoming_cash_order` - Приходный кассовый ордер (оплата)
- `customer_order` - Заказ клиента (предпродажа)
- `refund_sale_doc` - Возврат товаров от клиента (возврат)

У документов, содержащий табличную часть с товарами, помимо самого документа - `_header` (шапка), выгружается в дополнительная таблица: `tab_goods` (товары).


### Справочники (dimension)
- `organization` - Организации (UID, Наименование,	Ставка НДС)
- `employee` - Сотрудники (UID, ФИО, Подразделение)
- `product` - Номенклатура (UID, Код номенклатуры, Наименование номенклатуры, ставка НДС)


### Ограничения по данным:
- Дата документа продажи ≥ `2026-01-01` (начала текущего года).
- Для возвратов от клиента - отдельная логика по датам документов.
- Только проведённые документы.
- У документа "Чек ККМ" - статус = "Чек пробит".
- Номенклатура - только льготная ставка НДС (10).
