# "07.02 Документы с некорректной ставкой НДС."

## Описание задачи

## Бизнес-ситуация: 
Документы, передаваемые в ОФД (и ФНС), должны содержать корректную ставку НДС, согласно правилам организации.
**Правила применения ставки:**
- Ставка НДС документа =  актуальной ставке организации.
- Исключение: льготная ставка НДС (10 / 10/110) из номенклатуры, если у организации общая ставка (22)
- При возврате от клиента, ставка возврата = ставке исходного документа продажи (независимо от ставки организации).


### Цель DataMart:
Выявить все документы продажи/оплат/возвратов с ошибочными ставками НДС для контроля качества и исправлений.


### Описание DataMart 
Список ошибочных документов с указанием:
- даты и типа документа,
- наименования и ссылки на документ,
- категории ошибки,
- полей, подтверждающих ошибку,
- организации,
- ответственного (продавца) и его подразделения.


### Что считается ошибкой:
1. ставка НДС документа ≠ ставке НДС организации (в т.ч. Null/незаполнена),
2. ставка НДС документа ≠ льготной ставке номенклатуры, при общей ставки НДС организации (22),
3. ставка НДС возврата ≠ ставке НДС документа продажи.


## Используемые источники данных (Bronze / Raw)

### Документы (факты):
- `check_kkm` - Чек ККМ (продажа)
- `sale_doc` - Реализация товаров и услуг (продажа)
- `retail_sale_report` - Отчёт о розничных продажах (продажа)
- `payment_card_operation` - Эквайринговая операция (оплата)
- `incoming_cash_order` - Приходный кассовый ордер (оплата)
- `customer_order` - Заказ клиента (предпродажа)
- `refund_sale_doc` - Возврат товаров от клиента (возврат)

Каждый документ выгружается в 2-х таблицах: `_header` (шапка) + `_goods` (товары), кроме документов без табличных частей с товарами.


### Справочники (dimension)
- `organization` - Организации (UID, Наименование,	Ставка НДС)
- `employee` - Сотрудники (UID, ФИО, Подразделение)
- `product` - Номенклатура (UID, Код номенклатуры, Наименование номенклатуры, ставка НДС)


### Ограничения по данным:
- дата документа продажи ≥ `2026-01-01` (начала текущего года), для возвратов отдельный набор данных со своей логикой.
- только проведённые документы
- у "Чек ККМ" - статус = "Чек пробит"
- Номенклатура - только льготные ставки НДС (10, 10/110)
